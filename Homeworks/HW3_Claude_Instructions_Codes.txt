2. PHYSICS REMINDER — NONLINEAR SWE
================================================================================

PDE (nonlinear shallow water in conservative form):
  U_t + F(U)_x = 0

  U = [h; q]        state vector     (h = water height, q = hu = discharge)

  F(U) = [q; q^2/h + 0.5*g*h^2]    flux vector

Boundary conditions: REFLECTING WALL (q = 0) at both ends x=0 and x=L.
  → Ghost cell approach: mirror h, negate q at boundaries.

Domain: (0, L) × (0, T],   L = 1,  g = 9.81

No exact solution available → comparison between Godunov and Lax-Wendroff only.

────────────────────────────────────────────────────────────────────────────────

  Test a — Dam Break:

    h(x,0) = 1.0  if x < L/2,   h(x,0) = 0.5  if x >= L/2
    q(x,0) = 0    everywhere
    Expected: standing shock at interface

================================================================================
3. FVM NUMERICAL CORE — CONVENTIONS
================================================================================

Grid (uniform, M cells):
  dx   = L / M                          cell width
  x(i) = (i - 0.5) * dx,  i = 1..M    cell centers
  Interfaces at x_{i-1/2} = (i-1)*dx,  x_{i+1/2} = i*dx

State vector storage (2 × M matrices):
  U(1,:) = h(1..M)    row 1 = water height at all cell centers
  U(2,:) = q(1..M)    row 2 = discharge at all cell centers

CFL condition (MANDATORY — adaptive dt):
  c_max = max over all cells of ( |q/h| + sqrt(g*h) )
  dt    = CFL * dx / c_max        (CFL = 0.9 recommended)
  NEVER use a fixed dt. Recompute at every time step.

FVM update (conservative, explicit):
  U(:,i)^{n+1} = U(:,i)^n - (dt/dx) * ( F_{i+1/2} - F_{i-1/2} )

  where F_{i+1/2} is a 2×1 numerical flux vector at the right interface
  of cell i, computed by the chosen Riemann solver.

Reflecting wall BC (ghost cells, applied BEFORE flux computation each step):
  Left wall  (i=0  ghost): h_ghost = h(1),    q_ghost = -q(1)
  Right wall (i=M+1 ghost): h_ghost = h(M),   q_ghost = -q(M)

================================================================================
4. RIEMANN SOLVER — GODUNOV (1st order)
================================================================================

Assumption: piecewise constant reconstruction → U_L = U(:,i), U_R = U(:,i+1)

The exact Riemann solver for shallow water (Roe or exact):
  Given U_L = [h_L; q_L] and U_R = [h_R; q_R], compute the intermediate
  state U* and evaluate F(U*) at the interface.

Roe-average approach (simpler to implement, acceptable for HW3):
  u_L  = q_L / h_L,    c_L  = sqrt(g * h_L)
  u_R  = q_R / h_R,    c_R  = sqrt(g * h_R)

  Roe-averaged velocity:  u_hat = (sqrt(h_L)*u_L + sqrt(h_R)*u_R)
                                  / (sqrt(h_L) + sqrt(h_R))
  Roe-averaged celerity:  c_hat = sqrt(0.5*g*(h_L + h_R))

  Eigenvalues:  lambda_1 = u_hat - c_hat
                lambda_2 = u_hat + c_hat

  Flux (Roe upwind):
    F_roe = 0.5*(F(U_L) + F(U_R))
            - 0.5*(|lambda_1|*R_1*alpha_1 + |lambda_2|*R_2*alpha_2)

  where R_1, R_2 are the right eigenvectors of the Roe matrix and
  alpha_1, alpha_2 are the wave strengths (projection of dU = U_R - U_L).

ALTERNATIVE (simpler): Lax-Friedrichs numerical flux:
    F_LF = 0.5*(F(U_L) + F(U_R)) - 0.5*(dx/dt)*(U_R - U_L)
  This is more diffusive than Roe but correct for 1st order Godunov.
  Use Roe unless the assignment explicitly requires exact Godunov.

Function signature:
  function [Fnum] = GodunovFlux(U_L, U_R, g)
    % INPUT:  U_L (2x1), U_R (2x1), g (scalar)
    % OUTPUT: Fnum (2x1) numerical flux at the interface

================================================================================
5. RIEMANN SOLVER — LAX-WENDROFF (2nd order)
================================================================================

Step 1 — Linear reconstruction at both sides of interface x_{i+1/2}:
  U_L_recon = U(:,i)   + 0.5 * (U(:,i+1) - U(:,i))
  U_R_recon = U(:,i+1) - 0.5 * (U(:,i+1) - U(:,i))

  NOTE: as given by the homework statement (no slope limiter required).
  NOTE: these reduce to U_{i+1/2}^- and U_{i+1/2}^+ in the notation
        of the theory document.

Step 2 — Pass reconstructed states to the Riemann solver:
  Fnum = GodunovFlux(U_L_recon, U_R_recon, g)

  The same Riemann solver (Roe or LF) is reused — only the inputs change.

Function signature:
  function [Fnum] = LaxWendroffFlux(U_L, U_R, g)
    % INPUT:  U_L (2x1) = U(:,i), U_R (2x1) = U(:,i+1), g (scalar)
    % OUTPUT: Fnum (2x1) numerical flux at the interface

  Internally calls GodunovFlux with the reconstructed states.

7. MainFVM.m — FUNCTION SIGNATURE AND INTERNAL STEPS
================================================================================

  function [Solutions] = MainFVM(Data, scheme)

  scheme: 'godunov' or 'laxwendroff'

  Internal steps (in order):

    1.  Build uniform grid:
          dx = (Data.domain(2)-Data.domain(1)) / Data.M
          x  = cell centers (column vector, M×1)

    2.  Initialize state matrix U (2×M):
          U(1,:) = Data.h0(x)'
          U(2,:) = Data.q0(x)'

    3.  Time loop: t = 0, advance until t >= Data.T
          a. Compute local wave speeds: c_max = max(|u| + c) over all cells
          b. Compute adaptive dt = Data.CFL * dx / c_max
             Clip dt so that t+dt does not exceed Data.T
          c. Apply ghost cells (reflecting BC)
          d. For each interface i = 1..M+1 (including boundary interfaces):
               if scheme == 'godunov':    F_{i+1/2} = GodunovFlux(...)
               if scheme == 'laxwendroff': F_{i+1/2} = LaxWendroffFlux(...)
          e. Update:  U(:,i) = U(:,i) - (dt/dx)*(F_{i+1/2} - F_{i-1/2})
          f. t = t + dt

    4.  Store output:
          Solutions.x  = x
          Solutions.h  = U(1,:)'
          Solutions.q  = U(2,:)'
          Solutions.u  = U(2,:)' ./ U(1,:)'
          Solutions.t  = t

  NEVER plots inside this function.
  NEVER uses a fixed dt.
  NEVER inverts a matrix.

11. WHAT NOT TO DO
================================================================================

  ✗  Never use a fixed dt — always recompute from CFL condition each step
  ✗  Never invert a matrix — the FVM update is fully explicit
  ✗  Never call xwlgl, basis_and_der_at_lgl, or any SemLib function
  ✗  Never reference WAVE_1D/ or SEM infrastructure files
  ✗  Never plot inside MainFVM, GodunovFlux, or LaxWendroffFlux
  ✗  Never store intermediate time steps unless animation is required
  ✗  Never use a 2-char boundary string (e.g. 'NN') — BC is hardcoded
     as reflecting wall via ghost cells, not via a general BC dispatcher
  ✗  Never assume h > 0 without a safety check — add max(h, eps) where
     needed to avoid division by zero in u = q/h
  ✗  Never confuse dx (cell width = L/M) with Data.p (SEM degree, unused)
